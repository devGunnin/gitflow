- **[pitfall]** tests/e2e/pr_review_spec.lua persists PR review drafts to a REAL on-disk cache (stdpath('data')/gitflow/review/<repo-slug>/<pr>.json). Each test clears it only on the success path, so one failing test leaks a draft that review_panel.open() rehydrates - poisoning later tests in the same run AND every later run until the file is removed. A green suite can therefore turn red from an unrelated earlier failure. New tests should clear the PR's cache BEFORE open and locate drafts by content, not by pending_comments index.  _(2026-07-20T14:21:04Z)_
