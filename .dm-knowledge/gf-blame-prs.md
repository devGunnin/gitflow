- **[pitfall]** gh prerequisites are probed lazily on the first GitHub command, never during gitflow.setup() (startup must not pay a gh auth status round-trip). A test that asserts gh.state.checked right after setup() will fail — call gh.ensure_prerequisites() explicitly first.  _(2026-07-20T14:10:25Z)_
- **[convention]** Every async panel refresh guards its callbacks with a request_id generation counter (blame/prs/actions/rebase): bump on refresh/open/close, and drop any callback whose id is stale. Without it a superseded result clobbers the live panel — a reopened panel can render the previous file's blame or a dead error state.  _(2026-07-20T14:10:25Z)_
